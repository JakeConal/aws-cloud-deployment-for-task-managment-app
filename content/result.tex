\section{Kết quả và bằng chứng}

\subsection{Kết quả chức năng}

Hệ thống đã triển khai đầy đủ các chức năng chính của ứng dụng Task Manager:

\begin{itemize}
    \item Người dùng có thể đăng ký và đăng nhập bằng Cognito.
    \item Frontend truy cập được qua CloudFront.
    \item Direct S3 URL bị từ chối, chứng minh S3 bucket không public.
    \item Bốn endpoint CRUD hoạt động qua API Gateway REST API.
    \item API yêu cầu JWT token hợp lệ cho các endpoint \texttt{/tasks}.
    \item Dữ liệu công việc được lưu và truy vấn từ DynamoDB.
    \item Lambda chạy trong private subnet và truy cập DynamoDB qua VPC Gateway Endpoint.
\end{itemize}

\subsection{Chi phí}

Kiến trúc được thiết kế để giữ chi phí gần bằng 0:

\begin{itemize}
    \item Không sử dụng EC2.
    \item Không tạo NAT Gateway.
    \item Lambda có reserved concurrency để giới hạn invocation bất thường.
    \item API Gateway có throttling rate 100 request/giây và burst 50.
    \item DynamoDB dùng bảng và GSI trong phạm vi demo nhỏ.
    \item AWS Budget giới hạn 0.01 USD/tháng với cảnh báo 80\% và 100\%.
\end{itemize}

\subsection{Hạn chế và hướng phát triển}

Một số hạn chế hiện tại:

\begin{itemize}
    \item Ứng dụng chưa sử dụng custom domain.
    \item Quá trình triển khai được thực hiện thủ công qua AWS Console nên có thể khó lặp lại chính xác.
    \item Chưa có pipeline CI/CD tự động.
    \item Kiểm thử chủ yếu là kiểm thử thủ công qua frontend, curl và AWS Console.
\end{itemize}

Các hướng cải tiến:

\begin{itemize}
    \item Thêm custom domain và ACM certificate cho CloudFront.
    \item Bổ sung CI/CD để deploy frontend và Lambda tự động.
    \item Chuyển cấu hình hạ tầng sang SAM, CloudFormation, Terraform hoặc CDK trong phiên bản sau.
    \item Thêm kiểm thử tích hợp tự động cho API.
    \item Mở rộng chức năng tìm kiếm, phân trang và xác thực input.
\end{itemize}

\subsection{Kết luận}

Hệ thống đã đáp ứng mục tiêu của đồ án: triển khai ứng dụng web serverless trên AWS, bảo vệ frontend bằng S3 private và CloudFront OAC, bảo vệ API bằng Cognito, đặt Lambda trong private subnet, kết nối DynamoDB qua VPC Gateway Endpoint, áp dụng IAM least privilege, cấu hình monitoring và kiểm soát chi phí bằng AWS Budget. Thiết kế này đảm bảo khả năng tự mở rộng, tính sẵn sàng cao theo dịch vụ managed của AWS và hạn chế tối đa chi phí phát sinh.
